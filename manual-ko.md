## 1. 시작하기 {#getting-started}

### 1.1 회원가입 및 로그인 {#signup-login}
#### 이메일 가입 및 구글 소셜 로그인 연동
VibeX는 사용자의 편의를 위해 일반 이메일 가입과 Google 소셜 로그인 방식을 모두 지원합니다.

* **이메일 가입:** 로그인 화면 하단의 '회원가입'을 눌러 진행할 수 있습니다. 필수 약관에 동의한 후 발송되는 이메일 인증을 거치면 완료됩니다. 인증 메일이 보이지 않는다면 스팸 메일함을 확인하시거나 입력하신 이메일 앞뒤에 공백이 없는지 체크해 주세요.
* **Google 소셜 로그인:** 'Google로 로그인' 버튼을 클릭하여 간편하게 연동할 수 있습니다. 만약 버튼을 눌러도 흰 화면만 나온다면, 브라우저의 팝업 차단 기능이 인증 창을 막고 있을 수 있으니 팝업을 허용해 주시기 바랍니다.

![signup-login](https://cdn.vibe-x.app/images/ZDmjD2waTW-1773727221952.png)

> 💡 **접속 문제가 발생했나요?**
> 가입 버튼이 반응하지 않거나 무한 로딩이 발생할 경우, 브라우저의 광고 차단(AdBlock) 확장 프로그램을 잠시 끄시거나 시크릿 모드에서 접속해 보시는 것을 권장합니다.

### 1.2 계정 및 보안 설정 {#account-settings}
#### 프로필 관리, 비밀번호 재설정 및 데이터 파기
* **비밀번호 재설정:** 일반 이메일 계정의 경우 로그인 화면의 '비밀번호를 잊으셨나요?' 링크를 통해 비밀번호 재설정 메일을 요청할 수 있습니다. Google 소셜 로그인 사용자는 VibeX가 아닌 Google 계정 설정에서 직접 변경하셔야 합니다.
* **데이터 파기:** 개인정보 보호 정책상 파기된 데이터는 어떠한 경우에도 복구가 불가능하므로, 탈퇴 전 중요한 앱 코드는 에디터에서 개별적으로 내보내기(Export) 하여 백업해 두시기를 강력히 권장합니다.

### 1.5 크레딧 관련 주의사항

* 기본 제공 X 소멸: 매월 요금제로 제공되는 기본 X는 결제 갱신일에 초기화되며 다음 달로 이월되지 않으니 참고해 주세요.  
* 초과 사용(마이너스) 허용: 크레딧은 사용량에 따라 마이너스가 발생할 수 있습니다 (예: Pro 플랜의 월 최대 한도가 390 X인 경우, 사용량이 420 / 390 X처럼 한도를 초과할 수 있음). 이 상태에서 플랜을 업그레이드하면 초과된 사용량이 먼저 차감됩니다.  
* 추가 구매 X 우선 적용 및 이월: X 추가 구매로 충전된 크레딧은 요금제로 제공되는 크레딧보다 먼저 사용되며, 플랜 업그레이드 시 잔액이 그대로 이월됩니다.  
* 플랜 업그레이드 정산: 요금제 사용 중 플랜을 업그레이드할 경우, 기존 플랜에서 사용한 크레딧만큼 차감된 뒤 새로운 플랜 크레딧이 적용됩니다 (예: Pro → Ultra 업그레이드 시 90 X를 사용한 경우, 새 요금제 한도에서 차감되어 잔액 1,010 X 적용).

## 2. 앱 관리 및 대시보드 {#dashboard}
대시보드는 VibeX에 로그인한 후 가장 먼저 만나게 되는 메인 작업 공간입니다. 수많은 아이디어를 한곳에서 체계적으로 관리해 보세요.

### 2.1 대시보드 개요 및 탐색 {#dashboard-overview}
#### 내 앱 목록 확인, 검색 및 '중단한 곳에서 계속하기' 기능
* **중단한 곳에서 계속하기:** 홈 화면 > 앱 클릭 시 중앙에 위치한 영역으로, 최근까지 작업하던 앱 카드가 표시되어 클릭 한 번으로 빠르게 에디터로 복귀할 수 있습니다.
* **앱 목록 및 검색:** 상단의 '앱 검색...' 바를 이용해 특정 앱을 빠르게 찾을 수 있으며, 선호하는 앱(☆)을 등록하여 필터링할 수 있습니다.

### 2.2  앱 생성 및 관리 {#manage-projects}
#### 새 앱 만들기, 이름/설명 변경, 복제 및 삭제

* **새  생성:** 홈 화면 >  버튼 > 대시보드 우측 상단의 '+ 새 앱' 버튼을 클릭하여 백지상태에서 새로운 기획을 시작할 수 있습니다.
* **이름 및 정보 수정:**  앱 이름은 에디터 상단이나 설정에서 변경한 후, 반드시 Enter 키를 누르거나 빈 공간을 클릭해야 저장이 완료됩니다.
* **복제(Duplicate):** 기존 앱의 레이아웃이나 코드를 재사용하고 싶다면,  카드 우측 상단의 점 3개(더보기) 메뉴에서 '앱 복제'를 선택하여 똑같은 복사본을 만들 수 있습니다. 앱을 복제할 시 30 X가 차감됩니다. 
* **삭제(Delete):** 더보기 메뉴에서 '삭제' 버튼을 눌러 을 지울 수 있습니다.
    * **주의:** 한 번 삭제된 앱과 코드는 서버에서 즉시 영구 파기되어 절대 복구할 수 없으니 신중하게 결정해 주세요.

>**💡 앱 관리 팁** 현재 VibeX는 별도의 '폴더' 기능을 제공하지 않습니다. 앱이 많아져서 관리가 어렵다면, [쇼핑몰] 메인 화면, [포트폴리오] 연락처 폼처럼 대괄호를 활용해 이름 규칙을 통일해 두시면 검색과 관리가 훨씬 수월해집니다.


### 2.3 팀 워크스페이스 {#team-workspace}
#### 팀원 초대 및 권한 설정(편집자, 뷰어)을 통한 협업
혼자 작업하기 벅차다면 팀원들을 초대하여 하나의 앱을 함께 완성할 수 있습니다.

* **팀원 초대:** 앱 내의 공유버튼을 클릭한 후 동료의 이메일을 입력해 초대장을 발송할 수 있습니다.
* **권한 설정:** **권한 설정:** 초대받은 멤버는 관리자의 설정에 따라 자유롭게 코드를 수정할 수 있는 권한을 얻거나, 코드를 볼 수만 있는 '뷰어(Viewer)' 권한을 갖게 됩니다. 만약 내 앱이 '읽기 전용'으로 뜬다면 워크스페이스 관리자에게 권한 변경을 요청해 보세요.
* **소유권 제한:** 특정 앱의 소유권(Owner) 자체를 다른 사람의 계정으로 완전히 양도(이전)하는 기능은 현재 지원하지 않습니다.

## 3. AI 기반 앱 기획 및 생성 {#app-planning}
복잡한 다이어그램이나 기획서 없이, 오직 텍스트 프롬프트만으로 AI와 함께 앱의 뼈대를 잡고 코딩을 시작할 수 있습니다.

### 3.1 대화형 앱 기획 {#conversational-planning}
#### 텍스트 프롬프트를 활용한 UI 및 기능 구조 설계


* **프롬프트 입력:** 홈 화면 중앙의 텍스트 입력창에 만들고 싶은 앱의 아이디어를 자유롭게 적고 전송(종이비행기) 버튼을 누릅니다. (여러 줄을 입력하고 싶다면 Shift + Enter를 사용해 줄바꿈을 할 수 있습니다.)
* **명확한 지시:** 프롬프트가 짧고 추상적이면 결과도 일반적인 수준으로 나옵니다. 구체적인 타겟 사용자, 주요 컬러, 핵심 기능 등을 명확히 적어주실수록 훨씬 디테일한 기획안을 받아보실 수 있습니다.
* **플랫폼 지정:** VibeX는 웹 기반 생성이 기본입니다. 모바일 앱 형태를 원하신다면 프롬프트에 "반드시 모바일 화면 비율에 맞춘 UI와 터치 상호작용으로 기획해 줘"라고 명시해 주세요.

### 3.2 기획안 고도화 {#refining-plan}
#### 데이터베이스 스키마, API 목록, 마일스톤 추가 요청하기
기본적인 화면 구성 외에도 개발에 필요한 전문적인 명세들을 AI에게 추가로 요구할 수 있습니다.

* **구조화된 데이터 요구:** "이 앱에 필요한 전체 페이지 리스트와 각 페이지별 주요 기능을 표 형태로 정리해 줘"라고 요청해 보세요.
* **백엔드/API 기획안:** "데이터를 저장할 JSON 형태의 DB 스키마를 추가해 줘" 또는 "클라이언트와 서버가 통신할 REST API 엔드포인트 목록을 정의해 줘"라고 입력하시면 관련 기획을 덧붙여 줍니다.
* **컨텍스트 유지:** 대화가 너무 길어지면 AI가 초기에 지시한 중요한 내용(예: 특정 테마 적용)을 잊어버릴 수 있습니다. 이럴 땐 "처음에 말했던 다크모드 적용 규칙을 잊지 말고 반영해 줘"라고 다시 한번 상기시켜 주시는 것이 좋습니다.

### 3.3 앱 생성 파이프라인 이해 {#generation-pipeline}
#### 코딩 시작(Starting ~ Generating) 및 진행 상태 확인
기획 대화가 만족스럽게 마무리되었다면, 화면 우측의 **'앱 생성'** 버튼을 눌러 실제 코드를 만들어 낼 차례입니다. 생성은 다음 4단계를 거쳐 진행됩니다.

1. **Starting:** 앱 초기화 공간을 할당하는 단계입니다.  
2. **Setting up app:** 리액트(React)나 뷰(Vue) 등 환경 구성을 위한 필수 패키지를 서버에 설치합니다.  
3. **Thinking:** AI가 앱의 전체 구조와 컴포넌트 간의 관계를 설계합니다.  
4. **Generating:** 실제 코드를 작성하고 앱에 필요한 에셋(로고, 이미지 등)을 렌더링하는 최종 단계입니다.

>**💡 자동 복구(Auto-fix) 시스템** 생성 단계 진행 바가 갑자기 처음으로 돌아간 것처럼 보여도 놀라지 마세요! 생성 중 에러가 감지되면 AI가 스스로 코드를 수정하고 처음부터 다시 빌드를 시도하는 정상적인 복구 로직이 작동 중인 것입니다.

## 4. 에디터 및 실시간 미리보기 {#editor-preview}
코드가 성공적으로 생성되면 VibeX 에디터 화면으로 전환됩니다. 이곳에서 코드를 확인하고, AI와 대화하며 앱을 완벽하게 다듬을 수 있습니다.

### 4.1 에디터 인터페이스 {#editor-interface}
#### 코드, 데이터, 대시보드 탭 활용법
에디터는 크게 세 가지 탭으로 나뉘어 있습니다.
* **코드 탭:** 실제 화면을 구성하는 프론트엔드 소스 코드를 직접 확인하고 편집하는 공간입니다.
* **데이터 탭:** 앱에서 사용할 샘플 JSON 데이터나 백엔드 DB 구조를 확인하고 관리하는 곳입니다.
* **대시보드 탭:** 앱의 배포 상태를 확인하거나 환경 변수 설정, 도메인 연결 등 앱 운영 전반에 필요한 관리 설정을 수행합니다.


### 4.2 실시간 미리보기 및 로그 확인 {#preview-logs}
#### 우측 미리보기 렌더링 확인 
* **미리보기 화면:** 에디터 우측 영역에서는 코드가 렌더링된 결과를 실시간으로 확인할 수 있습니다. 브라우저 새로고침 시 앱 내부의 상태(예: 장바구니 담김, 로그인 상태)가 초기화되는 것은 웹 앱의 정상적인 작동 방식이니 참고해 주세요.



### 4.3 코드 수정 및 버전 관리 {#code-versioning}
#### AI 프롬프트로 코드 수정하기, 자동 저장 기능, 이전 버전으로 되돌리기
- **자동 저장(Auto-save):** VibeX 에디터는 기본적으로 코드 변경 사항이 감지될 때마다 백그라운드에서 실시간으로 자동 저장을 수행합니다. 단, 코드를 수동으로 편집한 직후 우측 미리보기가 갱신되기 전에 페이지를 새로고침하면 내용이 유실될 수 있으니 주의해 주세요. 
* **재생성 및 되돌리기:** AI에게 프롬프트를 입력해 코드를 재생성하면 기존 코드를 새로운 버전으로 덮어쓰게 됩니다. 만약 이전 결과물이 더 마음에 든다면 좌측 대화 기록(히스토리) 패널에서 '되돌리기' 기능을 사용해 과거 시점으로 코드를 롤백할 수 있습니다.

![revert](https://cdn.vibe-x.app/images/mNwz7eZ20b-1773818573662.png)

### 4.4 에셋 관리 

#### 이미지 파일 직접 업로드 및 교체 방법

AI가 생성한 이미지가 마음에 들지 않거나 내가 가진 로고를 넣고 싶다면 직접 에셋을 관리할 수 있습니다.

- **직접 업로드:** 좌측 하단 + 메뉴에 내 컴퓨터에 있는 이미지를 드래그 앤 드롭하여 쉽게 업로드할 수 있습니다.

- **용량 및 확장자 제한:** 쾌적한 환경을 위해 파일 1개당 최대 5MB까지만 업로드가 가능합니다. 웹 환경에 적합한 표준 이미지 형식(jpg, png, svg, webp)만 지원하며 보안상 실행 파일(.exe, .sh)은 업로드가 차단됩니다.

- **이미지 교체하기:** 이미지를 교체하는 방법은 다음 세 가지가 있습니다.
	1. 원하는 이미지를 **업로드한 후**, 해당 이미지로 **교체하도록 AI에게 요청합니다.**
	2. **AI에게 새로운 이미지를 생성하도록 요청한 후**, 생성된 이미지로 **교체를 진행합니다.**
	3. **이미지 주소(URL)를 복사하여** ‘코드(Code)’ 메뉴에서 수정하려는 파일에 **직접 붙여넣은 뒤 ‘저장’ 버튼을 클릭합니다.**

## 5. 프롬프트 작성 가이드 {#prompting-tips}

### 5.1 디자인 및 UI 최적화 {#design-optimization}
#### 컬러, 폰트, 다크모드, 모바일 반응형 디자인 적용 팁

* **정확한 컬러 및 수치 지정:** 배경색이나 포인트 버튼 색상은 `#1E1E1E`, `#FF5733`처럼 HEX 코드를 하드코딩해서 적용해 달라고 명시해 주세요. "모든 카드 UI의 `border-radius`는 16px, 내부 여백(Padding)은 24px로 넓게 잡아 줘"처럼 CSS 수치를 직접 제시하면 일관된 디자인을 얻을 수 있습니다.
* **다국어 처리 준비:** 다국어 확장을 고려한다면 "UI 텍스트를 하드코딩하지 말고 i18n 구조를 모방하여 한국어와 영어 문자열 객체를 따로 분리해 줘"라고 요청해 보세요.


### 5.2 복잡한 로직 구현 {#logic-implementation}
#### 관리자 페이지, 로그인/권한, 결제 모듈 추가 요청 방법
껍데기뿐인 UI가 아니라 실제 동작하는 로직을 원한다면 작동 흐름을 구체적으로 적어야 합니다.
* **로그인 및 상태 관리:** "이메일 폼을 만들고 로그인 상태를 관리하는 React State 로직을 반드시 포함시켜 줘"라고 데이터 상태 관리를 콕 집어 명시해야 합니다.
* **데이터베이스(DB) 구조:** "유저 정보와 게시글 데이터를 정의하는 JSON 형태의 데이터베이스 스키마(Mock Data)를 코드 상단에 구성해 줘"라고 요구하면 체계적인 데이터 구조를 짜줍니다.
- **별도 라우팅 및 관리자 페이지:** 일반 유저 화면 외의 기능이 필요할 땐 "상품 등록과 회원 관리를 할 수 있는 관리자 전용 대시보드 화면을 분리해서 별도의 라우팅으로 만들어 줘"라고 지시하세요.
- **결제 등 외부 모달 연동:** "장바구니에서 버튼을 누르면 Stripe 결제 모달이 뜨는 UI 흐름을 만들고, 가상의 결제 성공 Alert 창을 띄우는 함수를 연결해 줘"처럼 사용자 시나리오를 구체적으로 적어주세요.


### 5.3 효과적인 에러 해결 {#troubleshooting}
#### 빌드 실패나 렌더링 오류 시 코드 디버깅을 지시하는 방법
* **기존 코드 보존 강력 요청:** 특정 부분만 수정할 때는 "기존에 작성된 레이아웃과 로직은 100% 그대로 유지하고, 우측 상단에 알림 아이콘만 추가해 줘"라고 기존 상태 보존을 매우 강력하게 명령해야 디자인이 틀어지지 않습니다.
* **에러 로그 붙여넣기:** 우측 하단의 콘솔(Console)에서 붉은색 에러 텍스트를 드래그해 복사한 뒤, 챗봇 창에 그대로 붙여넣고 "이 에러를 고쳐줘"라고 요청하시면 AI가 원인을 파악해 수정해 줍니다.
* **원인 제시하기:** 단순한 "에러 났어"보다는 "패키지 충돌로 Build Failed 에러가 났어. 문제가 되는 라이브러리 import 구문을 수정해 줘"처럼 추정되는 원인을 함께 제시하면 복구 확률이 높아집니다.
* **결과가 엉뚱할 때의 대처:** AI가 지시와 전혀 다른 코드를 냈다면 그대로 덮어쓰기보다는, 히스토리에서 '되돌리기'로 코드를 원상 복구한 뒤 프롬프트 지시어를 더 뾰족하게 다듬어 다시 전송하는 것이 효율적입니다.

## 6. 백엔드 및 고급 연동 {#backend-integrations}
단순한 프론트엔드 UI를 넘어 실제 데이터를 저장하고 결제를 처리하는 등 강력한 앱을 완성하려면 외부 시스템과의 연동이 필요합니다.

### 6.1 백엔드 기능 활용 {#backend-functions}
#### 외부 기능 연결 및 API 호출
백엔드 기능(Backend Functions)은 서버에서 직접 실행되는 로직을 구현할 수 있게 해줍니다.

* **결제 모듈 연동:** Stripe나 포트원(Portone)과 같은 외부 결제 API를 백엔드 Functions를 통해 호출하여 구현할 수 있습니다. 실제 상용 결제를 붙이기 전에 반드시 테스트 모드로 충분히 검증하시기 바랍니다.
* **타임아웃 주의:** 외부 API를 과도하게 호출하거나 함수 내에 무한 루프가 돌 경우 타임아웃(Timeout) 처리되어 실행이 중단될 수 있으므로, 코드를 가볍게 유지하는 것이 좋습니다.


### 6.2 환경 변수 설정 {#env-variables}
#### API Secret Key 등 민감한 정보의 안전한 관리
외부 서비스를 연동할 때는 보안 설정에 각별히 주의해야 합니다.

* **시크릿 키 관리:** 민감한 키는 프론트엔드 코드에 직접 입력하지 말고 반드시 `[대시보드] > [환경 변수]`에 저장하세요.
* **환경변수 호출 확인:** 등록한 환경변수 키 이름과 코드 내부에서 호출하는 키 이름(예: process.env.VITE_API_KEY 등)의 대소문자가 정확히 일치하는지 꼼꼼히 확인해 주세요.
* **API 호출 제한(Rate Limit):** 남용 방지를 위해 단시간에 과도한 트래픽을 유발하거나 외부 API를 비정상적으로 호출할 경우, 시스템 자체 레이트 리밋에 의해 접근이 일시 차단될 수 있습니다.

![secret](https://cdn.vibe-x.app/images/e3MgKVwTS4-1773728743679.png)

### 6.3 GitHub 연동 {#github-integration}
#### 계정 권한 연결 및 앱 소스 코드를 내 Repository로 내보내기 (Push)
작성된 코드를 GitHub로 안전하게 내보내고 관리할 수 있습니다. (이 기능은 **Ultra 플랜 이상**부터 지원됩니다.)

* **Repository 생성:** GitHub 계정을 연동한 후, VibeX에서 코드를 내보낼 때 공개(Public) 또는 비공개(Private) 저장소 형식을 직접 선택하여 생성할 수 있습니다. 이름에 특수문자가 포함되거나 이미 같은 이름이 존재하면 생성이 거절될 수 있으니 영소문자와 하이픈(-) 조합을 권장합니다.
- **Push 권한 오류 해결:** VibeX와 GitHub 간의 보안 인증 토큰이 만료되어 Push 에러가 발생했다면, 앱 내의 코드 > Github 연결에서 GitHub 연동을 해제(Disconnect)한 후 다시 연결하여 권한을 갱신해 주시기 바랍니다.
- **브랜치 관리:** 현재 VibeX 내부에서는 main 브랜치 외에 다른 개발 브랜치(dev, feature 등)를 임의로 생성하거나 Pull Request(PR) 워크플로우를 진행하는 고급 Git 조작은 지원하지 않습니다.
- **충돌 방지:** 기존 GitHub 저장소에 VibeX가 모르는 다른 변경 사항(Commit)이 섞여 있으면 덮어쓰기가 거부될 수 있으므로, 가급적 빈 Repository를 새로 파서 연결하는 것을 추천합니다.

## 7. 배포 및 내보내기 {#publish-export}
멋지게 완성된 앱을 전 세계 유저들이 접속할 수 있도록 라이브 서버에 올리고, 필요하다면 원본 소스 코드를 다운로드해 보세요.

### 7.1 앱 배포하기 {#app-publishing}
#### 게시(Publish) 버튼을 통한 VibeX 호스팅 및 URL 발급
버튼 한 번으로 복잡한 서버 설정 없이 즉시 앱을 웹에 띄울 수 있습니다.

* **배포 시작:** 에디터 우측 상단에 있는 '게시(Publish)' 버튼을 누르면 현재 코드가 VibeX 호스팅 서버로 빌드 및 업로드되며, 누구나 접속할 수 있는 웹 주소(URL)가 발급됩니다.
* **배포 조건:** 백그라운드에서 AI가 코드를 아직 생성 중이거나 100% 저장되지 않았다면 게시 버튼이 비활성화되니, 생성이 완전히 끝난 후 시도해 주세요.
* **수정 사항 반영 (업데이트):** 에디터에서 코드를 수정하더라도 이미 배포된 라이브 서버에는 자동으로 반영되지 않습니다. 수정이 완료된 후 반드시 '게시(Publish)' 버튼을 다시 눌러 새 버전으로 덮어써 주셔야 실제 화면에 반영됩니다.
* **초기 접속 지연:** 배포 직후 발급된 URL로 접속했을 때 404 에러가 발생한다면, 전 세계 서버망(CDN)에 데이터가 전파되는 데 약간의 지연이 발생한 것입니다. 1~3분 정도 기다리신 후 브라우저 캐시를 지우고 다시 접속해 보세요.

![Publish](https://cdn.vibe-x.app/images/iGyPgbVegr-1773730818461.png)

### 7.2 커스텀 도메인 연결 {#custom-domain}
#### 내 도메인 DNS 설정 및 자동 SSL(HTTPS) 적용 방법
VibeX의 기본 주소 대신 `myapp.com`과 같은 나만의 멋진 도메인을 연결할 수 있습니다.

* **DNS 설정:** [대시보드] > [도메인] 메뉴에 소유하신 도메인을 입력하고, 화면에 제공되는 A 레코드 또는 CNAME 값을 도메인 구매처의 DNS 설정에 등록해 주세요.
* **전파 시간 대기:** DNS 설정을 변경한 후 전 세계 인터넷망에 이 정보가 전파되기까지 최대 24~48시간이 소요될 수 있으니 시간을 두고 기다려 주셔야 합니다.
* **SSL(HTTPS) 자동 발급:** 도메인 연결이 정상적으로 완료되면 VibeX에서 자동으로 보안 인증서(SSL)를 발급 및 적용해 드립니다. (단, 도메인 연결 직후에는 인증서 발급까지 1~2시간 정도 소요되어 브라우저에 '주의 요함'이 뜰 수 있습니다.)
* **www 리다이렉트:** `www.myapp.com`과 `myapp.com` 둘 다 원활하게 접속되게 하려면 DNS 설정에서 루트 도메인과 www 서브 도메인을 모두 등록해 주셔야 합니다.


### 7.3 소스 코드 다운로드 {#download-source}
#### 전체 앱 파일을 ZIP으로 내보내기 및 로컬 실행 방법
앱 코드를 완전히 소유하거나 Vercel 등 다른 외부 플랫폼에서 호스팅하고 싶을 때 유용합니다.

* **ZIP 다운로드:** 앱 내의 코드 > 앱 다운로드 아이콘을 클릭해 전체 소스를 다운로드하고, 로컬에서 실행할 수 있습니다.
* **로컬 환경 실행:** 다운로드한 코드를 내 컴퓨터에서 실행하려면, 앱 폴더 터미널에서 `npm install`을 실행하여 의존성 패키지를 설치한 후 `npm run dev` 명령어로 구동해 주세요.
* **외부 호스팅 배포(Vercel 등):** VibeX 내에서 직접 버튼 한 번으로 Vercel 등에 배포하는 연동은 지원하지 않습니다. 대신 코드를 본인의 GitHub 저장소로 Push(내보내기) 하신 후, 해당 저장소를 Vercel과 연결하여 배포하시는 것을 권장합니다.
