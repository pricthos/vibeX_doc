## 1. 시작하기 (Getting Started) {#getting-started}

### 1.1 회원가입 및 로그인 {#signup-login}
#### 이메일 가입 및 구글 소셜 로그인 연동
VibeX는 사용자의 편의를 위해 일반 이메일 가입과 Google 소셜 로그인 방식을 모두 지원합니다.
![img01](https://stg-cdn.vibe-x.app/apps/e61dbd754141778163e91e75/uploads/original/7a79f2c2-ad2c-4f36-9a84-007ec9588d03.png)

* **이메일 가입:** 로그인 화면 하단의 '회원가입'을 눌러 진행할 수 있습니다. 필수 약관에 동의한 후 발송되는 이메일 인증을 거치면 완료됩니다. 인증 메일이 보이지 않는다면 스팸 메일함을 확인하시거나 입력하신 이메일 앞뒤에 공백이 없는지 체크해 주세요.
* **Google 소셜 로그인:** 'Google로 로그인' 버튼을 클릭하여 간편하게 연동할 수 있습니다. 만약 버튼을 눌러도 흰 화면만 나온다면, 브라우저의 팝업 차단 기능이 인증 창을 막고 있을 수 있으니 팝업을 허용해 주시기 바랍니다.

> 💡 **접속 문제가 발생했나요?**
> 가입 버튼이 반응하지 않거나 무한 로딩이 발생할 경우, 브라우저의 광고 차단(AdBlock) 확장 프로그램을 잠시 끄시거나 시크릿 모드에서 접속해 보시는 것을 권장합니다.

### 1.2 계정 및 보안 설정 {#account-settings}
#### 프로필 관리, 비밀번호 재설정 및 계정 탈퇴
* **비밀번호 재설정:** 일반 이메일 계정의 경우 로그인 화면의 '비밀번호를 잊으셨나요?' 링크를 통해 비밀번호 재설정 메일을 요청할 수 있습니다. Google 소셜 로그인 사용자는 VibeX가 아닌 Google 계정 설정에서 직접 변경하셔야 합니다.
* **계정 탈퇴 및 데이터 파기:** `[설정] > [계정 관리]` 메뉴 하단의 '계정 탈퇴' 버튼을 클릭하면 계정과 모든 데이터가 완전히 영구 삭제됩니다. 개인정보 보호 정책상 파기된 데이터는 어떠한 경우에도 복구가 불가능하므로, 탈퇴 전 중요한 프로젝트 코드는 에디터에서 개별적으로 내보내기(Export) 하여 백업해 두시기를 강력히 권장합니다.

### 1.3 요금제 및 X 안내 {#pricing-and-x}
#### Pro, Ultra, Business, Enterprise 플랜별 기능 차이와 X 소모/복구
**X**란 VibeX 플랫폼 내에서 AI를 활용한 앱 기획, 코드 생성 및 수정 등 각종 사용량 기반 작업 시 소모되는 전용 재화를 의미합니다. VibeX는 사용 목적에 따라 4가지 플랜을 제공하며, 연간 구독 선택 시 **약 20%의 할인 혜택**을 받으실 수 있습니다. 매월 제공되는 기본 X는 결제 갱신일에 초기화되며 이월되지 않으니 참고해 주세요.

#### 1. Pro 플랜
* **월간 결제:** $39.00 / 월
* **연간 결제:** $374.40 / 연간 청구
* **제공 크레딧:** 100,000 X / 월
* **주요 특징:** 무제한 앱 생성과 인앱 코드 편집 기능을 기본 제공합니다. 가볍게 즐기거나 사이드 프로젝트를 테스트하기에 가장 합리적인 요금제입니다.

#### 2. Ultra 플랜
* **월간 결제:** $99.00 / 월
* **연간 결제:** $950.40 / 연간 청구
* **제공 크레딧:** 250,000 X / 월
* **주요 특징:** 본격적인 상용 서비스 운영을 준비하는 분들을 위한 플랜입니다. 백엔드 기능, 커스텀 도메인 연결, GitHub 연동 등 필수 고급 기능이 추가됩니다.

#### 3. Business 플랜
* **월간 결제:** $299.00 / 월
* **연간 결제:** $2,870.40 / 연간 청구
* **제공 크레딧:** 500,000 X / 월
* **주요 특징:** 잦은 수정과 방대한 작업량으로 많은 X 소모가 예상되는 팀 단위 작업에 추천합니다. 베타 기능 사전 접근 권한이 포함됩니다.

#### 4. Enterprise 플랜
* **월간 결제:** 협의형 요금제
* **연간 결제:** 협의형 요금제 
* **제공 크레딧:** 1,200,000 X / 월
* **구성:** * Tier 1: 월 3,000달러 연 28,800달러 (10명 기준)
    * Tier 2: 조직 규모에 따라 3,000달러 이상으로 별도 협의
    * Team 구성: 최소 5명 이상 기준
* **주요 특징:** 대규모 생성과 최고 수준의 지원이 필요한 팀을 위한 최상위 플랜으로, Business 플랜의 모든 기능에 최우선 기술 지원 혜택이 더해집니다.

## 2. 프로젝트 관리 및 대시보드 (Manage Projects & Dashboard) {#dashboard}

### 2.1 대시보드 개요 및 탐색 {#dashboard-overview}
#### 내 앱 목록 확인, 검색 및 '중단한 곳에서 계속하기' 기능
* **중단한 곳에서 계속하기:** 메인 홈 화면 중앙에 위치한 영역으로, 최근까지 작업하던 프로젝트 카드가 표시되어 클릭 한 번으로 빠르게 에디터로 복귀할 수 있습니다.
* **앱 목록 및 검색:** 상단의 '앱 검색...' 바를 이용해 특정 프로젝트를 빠르게 찾을 수 있으며, '내 앱 (개인 작업공간)' 드롭다운을 통해 워크스페이스별로 리스트를 필터링할 수 있습니다.

### 2.2 앱(프로젝트) 생성 및 관리 {#manage-projects}
#### 새 앱 만들기, 이름/설명 변경, 복제 및 삭제
* **새 프로젝트 생성:** 대시보드 우측 상단의 '+ 새 앱(또는 새 프로젝트)' 버튼을 클릭하여 백지상태에서 새로운 기획을 시작할 수 있습니다.
* **이름 및 정보 수정:** 프로젝트 이름은 에디터 상단이나 설정에서 변경한 후, 반드시 Enter 키를 누르거나 빈 공간을 클릭해야 저장이 완료됩니다. 프로젝트 설명은 입력 후 잠시 기다리면 자동 저장됩니다.
* **복제(Duplicate):** 기존 앱의 레이아웃이나 코드를 재사용하고 싶다면, 프로젝트 카드 우측 상단의 점 3개(더보기) 메뉴에서 '복제'를 선택하여 똑같은 복사본을 만들 수 있습니다.
* **삭제(Delete):** 더보기 메뉴에서 '삭제' 버튼을 눌러 프로젝트를 지울 수 있습니다.
    * **주의:** 한 번 삭제된 프로젝트와 코드는 서버에서 즉시 영구 파기되어 절대 복구할 수 없으니 신중하게 결정해 주세요.

> 💡 **프로젝트 관리 팁**
> 현재 VibeX는 별도의 '폴더' 기능을 제공하지 않습니다. 프로젝트가 많아져서 관리가 어렵다면, `[쇼핑몰] 메인 화면`, `[포트폴리오] 연락처 폼`처럼 대괄호를 활용해 이름 규칙을 통일해 두시면 검색과 관리가 훨씬 수월해집니다.

### 2.3 팀 워크스페이스 {#team-workspace}
#### 팀원 초대 및 권한 설정(편집자, 뷰어)을 통한 협업
혼자 작업하기 벅차다면 팀원들을 초대하여 하나의 프로젝트를 함께 완성할 수 있습니다.
* **팀원 초대:** 계정 설정에서 '팀 워크스페이스'를 생성한 후, 동료의 이메일을 입력해 초대장을 발송할 수 있습니다.
* **권한 설정:** 초대받은 멤버는 관리자의 설정에 따라 자유롭게 코드를 수정할 수 있는 권한을 얻거나, 코드를 볼 수만 있는 '뷰어(Viewer)' 권한을 갖게 됩니다. 만약 내 프로젝트가 '읽기 전용'으로 뜬다면 워크스페이스 관리자에게 권한 변경을 요청해 보세요.

## 3. AI 기반 앱 기획 및 생성 (App Planning & Generation) {#app-planning}

### 3.1 대화형 앱 기획 {#conversational-planning}
#### 텍스트 프롬프트를 활용한 UI 및 기능 구조 설계
* **프롬프트 입력:** 홈 화면 중앙의 텍스트 입력창에 아이디어를 자유롭게 적고 '시작하기'를 누릅니다. (`Shift + Enter`로 줄바꿈 가능)
* **명확한 지시:** 구체적인 타겟 사용자, 주요 컬러, 핵심 기능 등을 명확히 적어주실수록 훨씬 디테일한 기획안을 받아보실 수 있습니다.
* **플랫폼 지정:** VibeX는 웹 기반 생성이 기본입니다. 모바일 앱 형태를 원하신다면 프롬프트에 "반드시 모바일 화면 비율에 맞춘 UI와 터치 상호작용으로 기획해 줘"라고 명시해 주세요.

### 3.2 기획안 고도화 {#refining-plan}
#### 데이터베이스 스키마, API 목록, 마일스톤 추가 요청하기
* **구조화된 데이터 요구:** "이 앱에 필요한 전체 페이지 리스트와 각 페이지별 주요 기능을 표 형태로 정리해 줘"라고 요청해 보세요.
* **백엔드/API 기획안:** "데이터를 저장할 JSON 형태의 DB 스키마를 추가해 줘" 등을 입력하시면 관련 기획을 덧붙여 줍니다.
* **컨텍스트 유지:** 대화가 길어지면 "처음에 말했던 다크모드 적용 규칙을 잊지 말고 반영해 줘"라고 다시 한번 상기시켜 주시는 것이 좋습니다.

### 3.3 프로젝트 생성 파이프라인 이해 {#generation-pipeline}
#### 코딩 시작(Starting ~ Generating) 및 진행 상태 확인
생성은 다음 4단계를 거쳐 진행됩니다.
1. **Starting:** 프로젝트 초기화 공간을 할당합니다.
2. **Setting up project:** 리액트(React)나 뷰(Vue) 등 환경 구성을 위한 패키지를 설치합니다.
3. **Thinking:** AI가 전체 구조와 컴포넌트 간의 관계를 설계합니다.
4. **Generating:** 실제 코드를 작성하고 에셋을 렌더링하는 최종 단계입니다.

## 4. 에디터 및 실시간 미리보기 (Editor & Preview) {#editor-preview}

### 4.1 에디터 인터페이스 {#editor-interface}
#### 코드, 데이터, 대시보드 탭 활용법
에디터는 크게 세 가지 탭으로 나뉘어 있습니다.
* **코드 탭:** 프론트엔드 소스 코드를 직접 확인하고 편집하는 공간입니다.
* **데이터 탭:** 샘플 JSON 데이터나 백엔드 DB 구조를 확인하고 관리하는 곳입니다.
* **대시보드 탭:** 배포 상태 확인, 환경 변수 설정, 도메인 연결 등 앱 운영 설정을 수행합니다.

### 4.2 실시간 미리보기 및 로그 확인 {#preview-logs}
#### 우측 미리보기 렌더링 확인 및 콘솔(Console) 로그 디버깅
* **미리보기 화면:** 에디터 우측 영역에서 실시간 렌더링 결과를 확인할 수 있습니다.
* **콘솔(Console) 로그:** 우측 하단의 '콘솔' 버튼을 통해 빌드 에러나 자바스크립트 로그를 실시간으로 확인해 디버깅에 활용하세요.

### 4.3 코드 수정 및 버전 관리 {#code-versioning}
#### AI 프롬프트로 코드 수정하기, 자동 저장 기능, 이전 버전으로 되돌리기
* **자동 저장:** 코드 변경 시 실시간으로 자동 저장이 수행되나, 수동 편집 직후 갱신 전 새로고침 시 유실될 수 있으니 주의하세요.
* **재생성 및 되돌리기:** 새로운 코드가 마음에 들지 않는다면 좌측 대화 기록 패널에서 '되돌리기' 기능을 사용해 과거 시점으로 롤백할 수 있습니다.

### 4.4 에셋(이미지/폰트) 관리 {#asset-management}
#### 이미지/폰트 파일 직접 업로드 및 교체 방법
* **직접 업로드:** 좌측 파일 트리 메뉴에 파일을 드래그 앤 드롭하여 업로드합니다. (파일당 최대 5MB)
* **이미지 교체:** 에셋 업로드 후 코드 내의 이미지 경로(`src="..."`)를 파일 이름으로 변경하면 즉시 반영됩니다.

## 5. 프롬프트 작성 가이드 (Prompting Tips) {#prompting-tips}

### 5.1 디자인 및 UI 최적화 {#design-optimization}
#### 컬러, 폰트, 다크모드, 모바일 반응형 디자인 적용 팁
* **명확한 레퍼런스:** "Linear 앱 스타일의 미니멀리즘 테마로 만들어줘"처럼 대상을 명확히 제공하세요.
* **정확한 수치 지정:** 배경색 HEX 코드나 `border-radius: 16px` 같은 구체적인 CSS 수치를 제시하면 일관된 디자인을 얻을 수 있습니다.

### 5.2 복잡한 로직 구현 {#logic-implementation}
#### 관리자 페이지, 로그인/권한, 결제 모듈 추가 요청 방법
* **로그인 및 상태 관리:** "이메일 폼을 만들고 `isLoggedIn` 상태를 관리하는 로직을 포함해 줘"라고 명시하세요.
* **별도 라우팅:** "관리자 전용 대시보드 화면을 분리해서 별도의 라우팅으로 만들어 줘"라고 지시하세요.

### 5.3 효과적인 에러 해결 {#troubleshooting}
#### 빌드 실패나 렌더링 오류 시 코드 디버깅을 지시하는 방법
* **기존 코드 보존:** "기존 레이아웃은 유지하고, [특정 기능]만 추가해 줘"라고 강력하게 명령하세요.
* **에러 로그 활용:** 콘솔의 에러 텍스트를 복사해 붙여넣고 "이 에러를 고쳐줘"라고 요청하세요.

## 6. 커뮤니티 템플릿 (Community Templates) {#community-templates}

### 6.1 템플릿 탐색 및 미리보기 {#template-search}
#### 카테고리별 템플릿 검색 및 미리보기
* **검색 및 필터링:** 키워드나 카테고리 필터를 통해 원하는 템플릿을 찾으세요.

### 6.2 템플릿 가져오기(Import) {#import-templates}
#### 원하는 템플릿을 내 워크스페이스로 복사하여 수정하기
* **가져오기:** '가져오기' 버튼을 클릭하면 독립적인 복사본이 내 프로젝트로 등록됩니다. 
* **호환성 수정:** 과거 템플릿 실행 오류 시 "package.json 의존성 버전을 업데이트하고 고쳐줘"라고 요청하세요.

### 6.3 내 템플릿 공유하기 {#share-templates}
#### 내 프로젝트를 커뮤니티에 공개용으로 업로드하는 방법
* **공개 설정:** 프로젝트 설정에서 '공개' 및 '커뮤니티 공유' 옵션을 활성화하세요.

### 6.4 비용 및 라이선스 주의사항 {#template-license}
#### 템플릿 사용 시 주의해야 할 사항
* **X 차감:** 템플릿 가져오기 후 수정/재생성 시에는 본인 계정의 X가 정상 차감됩니다.
* **상업적 사용:** 원작자가 추가한 에셋은 저작권 문제가 있을 수 있으니 상업적 배포 전 본인의 에셋으로 교체하세요.

## 7. 백엔드 및 고급 연동 (Backend & Integrations) {#backend-integrations}

### 7.1 백엔드 기능(Functions) 활용 {#backend-functions}
#### 외부 DB(MongoDB, Supabase 등) 연결 및 API 호출
* **DB 연결:** 외부 DB 정보를 환경변수에 정확히 입력했는지 확인하세요.
* **결제 연동:** Stripe 등의 API를 백엔드에서 호출해 구현할 수 있습니다. 상용 전 반드시 테스트 모드로 검증하세요.

### 7.2 환경 변수(.env) 설정 {#env-variables}
#### API Secret Key 등 민감한 정보의 안전한 관리
* **시크릿 키 관리:** 민감한 키는 프론트엔드 코드에 직접 입력하지 말고 반드시 `[대시보드] > [환경 변수]`에 저장하세요.

### 7.3 GitHub 연동 {#github-integration}
#### 계정 권한 연결 및 프로젝트 소스 코드를 내 Repository로 내보내기 (Push)
* **Push 오류 해결:** 토큰 만료 시 GitHub 연동을 해제 후 다시 연결해 권한을 갱신하세요.
* **충돌 방지:** 가급적 비어 있는 신규 Repository에 연결하는 것을 추천합니다.

## 8. 배포 및 내보내기 (Publish & Export) {#publish-export}

### 8.1 앱 배포하기 {#app-publishing}
#### 게시(Publish) 버튼을 통한 VibeX 호스팅 및 URL 발급
* **배포 및 업데이트:** 수정 후에는 반드시 '게시' 버튼을 다시 눌러 새 버전으로 덮어써야 실제 화면에 반영됩니다.

### 8.2 커스텀 도메인 연결 {#custom-domain}
#### 내 도메인 DNS 설정 및 자동 SSL(HTTPS) 적용 방법
* **SSL 자동 발급:** 도메인 연결 후 VibeX에서 자동으로 보안 인증서를 발급합니다. (1~2시간 소요될 수 있음)

### 8.3 소스 코드 다운로드 {#download-source}
#### 전체 프로젝트 파일을 ZIP으로 내보내기 및 로컬 실행 방법
* **ZIP 다운로드:** 내보내기 아이콘을 클릭해 전체 소스를 다운로드하고, 로컬에서 `npm install` 후 실행할 수 있습니다.
